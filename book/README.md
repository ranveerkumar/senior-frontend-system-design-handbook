## What This Handbook Covers

This handbook is for engineers who already write good components and need to think bigger — about system design, platform architecture, failure handling, performance, security, and engineering leadership.

Ten chapters address the problems senior engineers most commonly encounter and most rarely have documented:

- How to design real-time systems that survive network disruption, reconnects, and optimistic state
- How to handle 500,000-row tables with proper query delegation, virtualization, and accessible semantics
- How to build schema-driven UI that is flexible without becoming unmaintainable
- How to classify and own state — local, server, URL, global, form, and workflow
- How to design performance as a structural property, not a post-release cleanup
- How to build a frontend platform that improves engineering velocity across teams
- How to architect failure handling with taxonomy, containment, retry, and observability
- How to reason about browser security boundaries, token exposure, RBAC, and CSP
- How to communicate system design clearly in senior engineering interviews
